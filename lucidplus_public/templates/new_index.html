GitHub Pages File Location Guide

Where to place your files for proper GitHub Pages deployment

Understanding the Problem

You're seeing a 404 error because GitHub Pages cannot find a valid index.html file in the correct location.

GitHub Pages looks for files in specific locations depending on your site type:

User/Organization Sites

Repository name: username.github.io

File location: Root of the repository

  • index.html
  • styles.css
  • script.js

URL: https://username.github.io

Project Sites

Repository name: Any name

File location: Root or /docs folder

  • index.html
  • css/styles.css
  • js/script.js

URL: https://username.github.io/repository-name

Step-by-Step Solution

1. Check Your Repository Type

Determine if you're creating a User/Organization site or a Project site:

  • User/Organization site: Repository must be named username.github.io (replace "username" with your GitHub username)
  • Project site: Repository can have any name, and files can be in the root or in the /docs folder

2. Place Files in the Correct Location

Based on your repository type, make sure your files are in the right location:

For User/Organization sites:
username.github.io/
├── index.html
├── css/
│ └── style.css
├── js/
│ └── script.js
└── images/
└── logo.png

For Project sites (root deployment):
repository-name/
├── index.html
├── css/
│ └── style.css
├── js/
│ └── script.js
└── README.md

For Project sites (/docs deployment):
repository-name/
├── docs/
│ ├── index.html
│ ├── css/
│ │ └── style.css
│ └── js/
│ └── script.js
└── README.md

3. Enable GitHub Pages

Make sure GitHub Pages is enabled in your repository settings:

  1. Go to your repository on GitHub
  2. Click on the "Settings" tab
  3. Scroll down to the "GitHub Pages" section
  4. Select the source branch (and folder if using /docs)
  5. Click "Save"

Note: If you're using the /docs folder option, make sure your files are in the /docs folder at the root of your repository, not in a subfolder.

4. Check File Names and Case Sensitivity

GitHub Pages is case-sensitive. Make sure:

  • Your main page is named index.html (all lowercase)
  • All file references in your HTML use the correct case
  • There are no spaces or special characters in your file names

Best Practices

  • ✓ Use relative paths for all internal links (e.g., "css/styles.css" instead of "/css/styles.css")
  • ✓ Keep all filenames lowercase to avoid case sensitivity issues
  • ✓ Use a simple structure without complex nesting
  • ✓ Test your site locally before pushing to GitHub

Deployment Time: After pushing your changes, it might take 1-10 minutes for GitHub Pages to deploy your site. You can check the deployment status in the "Environments" section of your repository.
